    The Terraform(TF) Kubernetes Provider¬§

    This is the terraform tool to connect to a running kubernetes cluster.

    Requirements¬§

    • The kubernetes cluster we created on DO.
    • The completed generic k8s tutorial, with the container image for the app on our private registry.
    $ kubectl get nodes

    Configuration¬§

    When we did set up the kubernetes cluster on DigitalOcean(DO), we pulled the K8s config, incl. certificates, into our local ~/.kube/config (using the cloud infra specific tool doctl: doctl kubernetes cluster kubeconfig save "$clustername").

    That file is being used by the TF K8s provider.

    Note

    There are other means to connect as well. Here we show how to configure the kubernetes provider with only the cloud provider's API token and no kubectl config file.

    Create Namespace¬§

    Here a basic first use case: Lets create a namespace on K8s, using TF:

    $ cat namespace.tf

provider "kubernetes" {
  config_path    = "~/.kube/config"
}

resource "kubernetes_namespace" "tfprov" {
  metadata {
    name = "tfprov-namespace"
  }
}

    $ terraform init
$ terraform apply -auto-approve # lp: asserts="Apply complete"
